Defines the fixed set of tags for SecretLead's bundled default tag file — the one seeded into AppData automatically on first run, and the one used to build the Construction demo (README.md §11). Its `##` line, **Construction**, specifies the entity name, which is also its record-store filename (`Construction.txt`). A CSV file is importable only if its header row contains one column per tag below, matched by name (case-insensitive, any column order) — extra or missing columns block the import and are named explicitly in the Status flyout, not silently skipped (README.md §9). The first tag listed plays the "Category" role (top-level grouping, and the field exploded into multiple records when semicolon-delimited on import); the second tag listed plays the "Nickname" role (searchable business identifier).

This file ships as `.\assets\construction.md` — a read-only packaged asset. There is no folder picker and no folder-as-project model (README.md §3) — Import uses two independent file pickers instead. At Import time, whichever tag file was picked (this one or any other) is copied into AppData as `<EntityName>.md`, alongside the `<EntityName>.txt` it produces (README.md §5). On first run, when AppData has no entities yet, this file and its matching `construction.txt` sample are seeded in automatically the same way (README.md §6, §11), so this copy is never edited or written to directly by the running app either way.
